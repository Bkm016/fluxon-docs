<Note>字段定义与 JSON 结构以 [函数目录](/appendix/function-catalog) 为准（canonical）。本页只描述“刷新与同步”的工作流。</Note>

## 工作流

1. **导出目录**：在仓库根目录运行 `./gradlew :core:dumpFluxonCatalog`（默认输出 `core/build/fluxon-functions.json`）。
2. **同步到 VS Code**：在 `vscode-extension` 目录运行 `pnpm run sync-catalog`（详见 [VS Code 扩展](/tooling/vscode)）。
3. **更新文档**：目录字段结构有变化时只更新 [函数目录](/appendix/function-catalog)；只是函数新增/改名/参数变化时更新对应讲解页，
   并确保示例与导出的目录一致。

## 何时执行

- 新增/删除/重命名运行时函数或扩展函数。
- 修改参数个数、命名空间、`async`/`primarySync` 属性。
- 发布 VS Code 扩展或更新文档前。
